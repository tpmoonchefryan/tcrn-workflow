# TCRN Workflow 0.1.0-rc.5

`0.1.0-rc.5` is an immutable unpublished local Workflow release candidate. It
lands the post-MVP hardening program on the two officially supported V1 Agent
Apps — Codex and Claude Code — while keeping the public protocols host-neutral.
The prior `0.1.0-rc.1`, `0.1.0-rc.2`, `0.1.0-rc.3`, and `0.1.0-rc.4` candidates
remain immutable and are superseded for current-candidate purposes.

Shipped surface in this candidate:

- **Governed conference and gate surface**: nine governed CLI verbs
  (`conference-open`/`-append-position`/`-close`/`-cancel`,
  `gate-create`/`-transition`/`-delete`, `conference-list-by-work`, `gate-list`)
  persist `conference.*`/`gate.*` records as additive hash-chained workspace
  events through the single SDC-2 payload constructor, under a held lease and the
  engine `expectedVersion` CAS, with byte-identical views/export/archive for
  workspaces that carry no extension events.
- **Fail-closed decision gates**: a non-tombstoned pending gate anchored to a
  work item blocks that item's transition to `done` with `WORKSPACE_GATE_PENDING`
  at the verb and identically on replay, and a gate reaches `satisfied` only
  against a resolvable `conference-minutes` locator; the frozen work status graph
  is unchanged.
- **Actor-attestation enforcement**: appending the one-way
  `attestation.actor.enabled` chain event makes a valid actor id mandatory from
  that sequence onward on both the live append path and the replay reducer
  (`WORKSPACE_ACTOR_REQUIRED` / `WORKSPACE_ACTOR_INVALID`); a workspace that never
  enables it stays byte-identical to `0.1.0-rc.4`.
- **Three-step Claude Code activation ladder** as gated, byte-reversible
  capability: the Step-1 installer writes the four inert bundle templates under
  `.claude/tcrn-workflow/` (O_EXCL/O_NOFOLLOW, never touching
  `.claude/settings.json`), Step 2 merges exactly one fail-open `SessionStart`
  hook, and Step 3 renders the single advisory Verity persona authority summary
  within the 1024-byte injection budget; nothing under `~/.claude` is ever named
  or written and every step is an exact byte-inverse on rollback.
- **Snapshot backup and hermetic restore**: a lease-held `snapshot-manifest`
  emits a deterministic per-file manifest, `snapshot-verify` reports
  `SNAPSHOT_VERIFIED`/`SNAPSHOT_MISMATCH`, the restore runbook round-trips
  snapshot → wipe → restore byte-identically at the original path with both
  doctrine failure modes failing closed, and an optional git tier-2 serves as an
  integrity witness only.
- **Knowledge Core extensions**: capture-cheap `knowledge-create`, governed
  `knowledge-rebase` head re-binding (`KNOWLEDGE_REBASE_BLOCKED`), the
  reverify/retire lifecycle under CAS (`KNOWLEDGE_LIFECYCLE_INVALID`),
  unconditional promotion governance (`KNOWLEDGE_PROMOTION_INVALID`), and
  close-time conference distillation of each minutes decision into a backlinking
  knowledge candidate.
- **O(delta) engine**: the quadratic replay cost is removed — an n-event chain
  runs exactly one terminal full-graph validation plus one ancestor-bounded
  O(delta) closure per work event, proven by closed-form operation-count
  equality.
- **Agent-integration consumption contract**: envelopes, the retry table for
  `WORKSPACE_VIEW_STALE`/`WORKSPACE_LOCKED`/`WORKSPACE_CAS_MISMATCH` (plus the
  lease verbs, `WORKSPACE_GATE_PENDING`, and `SNAPSHOT_*` codes), the
  `-`/`null`/`head` sentinels, and determinism guarantees, with a drift-guard
  test binding the prose to the live command catalog.
- **Advisory time-attestation receipts** via `--attest-dir` on every
  workspace-event mutation verb: the engine reads no clock, receipts write only
  outside the workspace root, embed no path or hostname, and carry no governance
  weight; export and archive bytes are identical whether or not receipts exist.
- **Settings-catalog knobs, spec expansion, and the proof-to-product budget
  rule**: the `settings-catalog-v1` conference and backup knobs are documented,
  the one-page protocol stub specs are expanded to normative weight, and the
  proof-to-product budget rule is recorded as a reviewer-enforced `CONTRIBUTING`
  policy.
- **Flagship end-to-end governed loop**: one hermetic replay of initiative →
  epic → story → gate → conference → distill → promote → trace on a real
  workspace, every tutorial command executed verbatim and every produced digest
  traced to its producer (`pnpm verify:e2e`).
- **`docs/architecture/rc5-compatibility.md`**: the rc.4 → rc.5 workspace
  compatibility and migration matrix (intentional forward-incompatibility of
  conference/gate/attestation events under `storageVersion 1`, the one-way
  attestation boundary, and the disposable knowledge-store re-initialization
  procedure).

This candidate is not published, is not a supported AOS release, has
`supportedAosReleases: []`, and does not authorize remote, AOS, connected-mode,
or publication operations. No live Codex or Claude Code host support is asserted;
both adapters remain inert dry-run candidates.
