# TCRN Workflow 0.4.0

`0.4.0` is the sixth accepted release. It adds background-resource residue
governance — the machine-checkable half of a convention that keeps a governed
agent session from leaking the background loads it spawns
(`TCRN-CROSS-INIT-007`, EPIC-014).

## Background-resource residue detection (`TCRN-CROSS-INIT-007`, EPIC-014)

A governed session that spawns a background load — a CPU-stress loop, a dev
server, a headless browser, a watcher — owns it for the session and must reclaim
it at teardown. When it does not, a detached child outlives the shell that led
its process group, reparents to init, and burns the host silently. The
originating incident: five CPU-stress groups whose shell leaders exited left 35
`yes` orphans at roughly seven cores for about five hours before anyone noticed.

- **A pure, deterministic core (`packages/core/src/background-resource.ts`).** A
  registration face records the process **group** a session owns — the pgid is
  the stable handle, since pids are reused and orphans outlive their leader — as
  canonical JSONL. A detection face, given a registration set and a
  process-table snapshot, reports residue: any owned group with a live member,
  or any orphan (reparented to init or whose parent has left the snapshot) whose
  command matches a registered pattern. No `ps`, no `fs`, no `Date`, no
  randomness — it is fixture-testable, and its red-proof (guard-registry `BR-01`,
  `BR-02`) is that the injected orphan is always detected.
- **A thin host adapter (`scripts/spawn-guard.mjs`).** `register` / `deregister`
  / `list` / `detect`, reading the live table with the reaper's hardened `ps`
  invocation and storing the registry in the workspace transient zone — outside
  the engine control tree, so replay and the snapshot witness never see it.
  `detect` prints a canonical JSON residue report and exits `0` clean, `3`
  residue present, `1` on its own failure.
- **Host wiring, stated without overclaim
  (`docs/architecture/background-resource-governance.md`).** The detector is
  invokable on demand today on any host. Automatic session-end firing is *not*
  activated: Claude Code's `Stop` hook is a signed-ladder-step decision (it is a
  KEEP-INERT ladder non-goal), and Codex's is trust-gated and pending a
  session-end-event survey. Both are recorded as recipes, not as done work.

The engine and the event protocol are unchanged: the registry lives in the
transient zone and nothing here writes the control tree, so a workspace that
never uses the detector is byte-identical to `0.3.2`.

## Verification

`verify:p1` (20 gates) and `verify:p8` are green, `guard-check` proves every
registered red-proof still bites (including the two new detector guards), and
the new capability ships with `tests/background-resource.test.mjs` and
`tests/spawn-guard.test.mjs`.
