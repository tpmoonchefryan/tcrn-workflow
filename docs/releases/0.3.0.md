# TCRN Workflow 0.3.0

`0.3.0` is the third accepted release. It does one thing: it lets a work item
carry its own authoritative scope. Until now a Story or an Epic was an external
key, a kind, a status, and a place in the graph — its *meaning* lived only in
the conference that decided it, and an agent that read the label instead of the
decision could get the scope wrong. `work-annotate` closes that gap by writing
the scope, and a backlink to the deciding minutes, onto the record itself.

The minor version is honest about a new operation, not a behaviour change: no
existing verb behaves differently, and a workspace that never annotates is
byte-identical to one under `0.2.0`. Chains written under `0.2.0` replay
unchanged.

## Advisory fields, and why they are non-binding

`work-annotate` takes `--scope` (an authoritative one-line scope/intent
statement) and `--decided-by` (a list of `minutes:` ids that govern the item).
Both land in the record's extensions as `advisory:scope` and
`advisory:decided-by`, each a `required:false` entry — the same shape the gate
evidence and gate identity decisions already use, which needs no registry row.
They are deliberately non-binding: an annotation never changes status, never
satisfies or blocks a gate, and never affects whether the item can reach
`done`. It is a record of intent, checked for shape, not a control.

`work-show` surfaces the advisory fields under an `advisory` key when they are
present, and omits it — byte-for-byte as before — when they are not.

## A new operation, and the boundary it draws

Annotation is its own event operation, `work.annotated`, not a `work.updated`:
the transition table has no self-loop, so an annotation cannot masquerade as a
status change, and keeping it separate leaves the transition and gate-clearance
paths untouched. That makes it a WSD-1 additive operation, exactly like the
conference and gate operations before it: a workspace that uses it becomes
unreadable by a binary that predates it, while `storageVersion` stays 1 and
every workspace that never annotates stays fully readable.

The reducer replays a `work.annotated` event only when it changes exactly the
two advisory keys and leaves every other field — status above all — and every
other extension byte-identical. A hand-crafted annotation that smuggles a status
change or a foreign extension past that guard fails closed
`WORKSPACE_EVENT_CORRUPT`, pinned by three forge tests.
