# TCRN Workflow 0.1.0

`0.1.0` is the first accepted release. Semantic Versioning applies from here;
in the 0.x range the public API may still change between minor versions. The
publication is the Owner-signed annotated tag `v0.1.0` and the GitHub Release
that carries the deterministic P8 artifacts; the companion helper accepts the
bytes through its independently published bootstrap digest.

Every release candidate before this one was an immutable local cut whose
manifest said so: `unpublished_candidate`. This one says `accepted_release`,
and the difference is not ceremony. Between `rc.6` and `0.1.0` the program did
one thing: it replaced assumptions with observations, and rewrote every public
claim to match.

## What was measured, not asserted

**Claude Code activation is live and was observed against a real host.** The
`pnpm host-evidence` harness drives install, activation and removal against
Claude Code `2.1.201`: eight credential-free observations, plus a credentialed
readback in which the model quoted a per-preparation nonce from the injected
authority summary — proving the summary reaches the model's context rather
than merely landing on disk. Receipt: `docs/verification/host/claude-code.json`.
What remains unbuilt is the operator command path: from a shell,
`claude-adapter-install` stops at `ADAPTER_HOST_REQUIRED`, and the README says
exactly that.

**The scale ceiling has a number.** A workspace slows perceptibly in the low
thousands of events and a single command crosses one second around ~6,600
(Apple M3, extrapolated; raw samples in
`docs/verification/2026-07-20-event-chain-ceiling-samples.json`). Reads pay the
same price as writes. The shipped guidance is the only mitigation that exists:
partition workspaces per project, not per organisation.

**The documentation now carries its limits.** All five READMEs state the four
permanent design boundaries, twelve operational Known limits — each enforced by
a reason code, pinned by a measurement, or declared untested and therefore
unclaimed — and a Driver assumptions group: integrity does not depend on the
driving model, because fail-closed turns a weak driver's boundary misses into
refusals; what scales with capability is progress. No compatible-model list is
published, because only one configuration was ever measured.

## Byte-contract hardening since rc.6

- `canonicalDocumentBytes` names the text-plus-trailing-newline contract that
  document digests rely on; the release-trust signature basis is unchanged.
- The two strict RFC 3339 parsers are pinned to one another by a shared
  accept/reject corpus with nanosecond equality.
- The recovery stress test now asserts the implemented promise — at most one
  winner, and liveness after every round — after CI produced a legitimate
  zero-winner run on bytes that later reran green.
- `pnpm guard-check` turns "revert the guard and watch its test redden" into a
  machine judgement over a twelve-entry registry.

## What this release does not claim

- `supportedAosReleases` is empty: no AOS pair, no connected mode, no AOS
  mutation.
- Twelve governed verbs (profile admission, context routing, compatibility
  planning, adapter families) have no operator entry point; the shipped CLI
  supplies none of the out-of-band authorities they require.
- One OS user on a local filesystem is the tested envelope. Cross-user sharing
  and network filesystems are untested and therefore unclaimed.
- Restore is same-path-only, and one writer per workspace is the concurrency
  model. Parallelism belongs above the storage layer.
