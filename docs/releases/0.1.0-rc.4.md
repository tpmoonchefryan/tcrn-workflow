# TCRN Workflow 0.1.0-rc.4

`0.1.0-rc.4` is an immutable unpublished local Workflow release candidate. It
re-cuts the MVP scope to two officially supported V1 Agent Apps — Codex and
Claude Code — while keeping the public protocols host-neutral, and adds four
additive extension work shapes. The prior `0.1.0-rc.1`, `0.1.0-rc.2`, and
`0.1.0-rc.3` candidates remain immutable and are superseded for current-candidate
purposes.

Scope additions in this candidate:

- **Claude Code adapter (P6B)**: an inert, uninstalled, unactivated Claude Code
  adapter emitting a four-file `.claude/tcrn-workflow/` template bundle bound to a
  `claude-code` host identity and a data-only version readback, a byte-reversible
  `.claude/settings.json` settings hook fragment that never clobbers user content,
  forbidden user-level `.claude` path rejection, and a CLAUDE.md-only fallback. A
  cross-host parity proof byte-binds the Codex adapter; the Codex adapter is
  unchanged.
- **`dependency-v1`**: an additive work-dependency extension record (blocks /
  informs; active / resolved / waived with a fail-closed waived audit; canonical
  hashing; deterministic ordering; live same-project endpoint rules with
  cross-project and self-edge fail-closed; active-blocks cycle detection;
  list-blockers).
- **`conference-v1`** (skeleton): pre-commitment deliberation records (request /
  position / minutes) with a minimal operation surface and close-distillation into
  knowledge decision candidates.
- **`assignment-v1`** and **`gate-v1`** (skeletons): a work-to-actor accountability
  binding and a minimal decision gate grounding knowledge `linkedGateIds`.
- Documentation: a `work-log-v1` dispatch-closeout convention and a
  `settings-catalog-v1` conversational-settings catalog.
- Public requirement ledger: appended `AOS-REQ-015..019` (now 19 entries) and
  flipped `AOS-REQ-006` and `AOS-REQ-016` to `fixture_verified`.

`work-model-v1`, `codex-adapter.ts`, and the generic starter path are unchanged.
This candidate is not published, is not a supported AOS release, has
`supportedAosReleases: []`, and does not authorize remote, AOS, connected-mode,
or publication operations. No live Codex or Claude Code host support is asserted;
both adapters remain inert dry-run candidates.
