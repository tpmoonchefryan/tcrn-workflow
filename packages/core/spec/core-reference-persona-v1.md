# Core Reference Persona V1

This bounded P5 surface publishes exactly eight inert public reference records:
Arturo, Mara, Minerva, Ilya, Verity, Sable, Janus, and Mneme. The sole semantic
source is admission manifest SHA-256 `9fa68e8f06e73e1d1b4bffb59a059814e683619b1d80234aef82e44f76de7c13`.
Each profile is also bound to its exact frozen semantic projection; recomputed
profile and bundle digests cannot admit changed semantics.

Records are closed, canonical, deterministic data. They contain identity,
mission, authority boundary, contact conditions, inputs, deliverables,
refusals, success criteria, and stable collaboration names only. They contain
no current state, routes, thread/session identifiers, model settings, private
facts, paths, credentials, runtime/release state, executable tags, hooks,
Skills, network/database/AOS authority, or incident history.

Generated release-profile layers are display-only and remain unbound and
non-authorizing. The generic-profile external path/file-digest admission anchor
and explicit owner rebind remain mandatory. No live profile store is created.

Structural schema/runtime limits are identical and count Unicode code points,
including each non-BMP scalar as one: job titles are 2..128 code points; list
fields contain 1..16 unique items of 2..256 code points; mission, authority, and
contact text are 10..512. Exact-source admission is a separate
stricter check. The property proof executes 64 distinct input orders and
normalizes each through the exact validator to one accepted bundle.
